# Contributing to AVAIL Tips and Tricks

Thanks for contributing. This repository is designed to stay practical, lightweight, and easy to use.

## Before you start

1. Check `docs/Home.md` and `docs/getting-started.md`.
2. See whether similar content already exists.
3. Prefer extending existing pages before creating duplicates.

## What you can contribute

- A tutorial or walkthrough (`tutorials/`)
- A notebook example (`notebooks/`)
- A small reusable script (`scripts/`)
- A “Getting started with X” guide (`docs/`)
- A checklist or reference (`references/`)

## Lightweight contribution workflow

### Proposing a tutorial

- Open an issue using the content proposal template.
- State: purpose, audience, prerequisites, and expected time to complete.
- Once agreed, create content using `templates/tutorial-template.md`.

### Submitting a notebook or script

- Add a short README/description with purpose, environment, and expected output.
- Keep dependencies minimal and clearly listed.
- Include one realistic example of use.

### Adding a “Getting started with X” guide

- Use `templates/getting-started-guide-template.md`.
- Focus on first successful use in 10–20 minutes.
- Include links to official docs and one AVAIL-relevant example.

## Quality expectations

Keep content:
- concise;
- practical;
- reproducible;
- written in clear British English.

## Pull request checklist

Before opening a PR, confirm:
- [ ] file is in the right folder;
- [ ] title and purpose are clear;
- [ ] prerequisites/dependencies are explicit;
- [ ] steps are runnable by someone new to the tool;
- [ ] links and paths are valid.

For full documentation expectations, see `docs/standards-documentation-quality.md`.
